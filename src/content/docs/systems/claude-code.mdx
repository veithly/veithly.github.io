## §1 · Claude Code 是什么

Anthropic 出品的 agentic coding tool，以 npm 包 `@anthropic-ai/claude-code` 发布。同一份 TypeScript 代码库支持终端、Cursor 风格 IDE 插件、Web、Mobile 四种入口。

## §3 · 工程亮点

1. **skill 机制是行业天花板**：内置 17 个 skill 加 `skillify` 元 skill。`skillify` 用 4 轮 AskUserQuestion 把一段 session 蒸馏成 SKILL.md，是「agent 自己产 skill」最完整的闭环。
2. **多入口共用一份代码**：4 种入口走同一个 turn loop，只在 `entrypoints/` 层做协议适配。
3. **autoMode classifier**：让 LLM 审用户写的 auto allow、soft_deny、env 分类规则，从 clarity、completeness、conflicts 三个角度评。
4. **`/security-review` 当 senior security engineer**：LLM 跑 PR diff，5 类漏洞加 80% 置信度门槛，主动排除 DOS 和磁盘密钥这类问题。
5. **MEMORY_DRIFT_CAVEAT 显式承认 stale**：prompt 里直接告诉模型记忆可能过时，要求先 grep 验证再用。

## §4 · 短板

1. **闭源**：所有结论基于 npm 包反编译，部分行为只能推断。
2. **没有跨平台原生 sandbox**：依赖宿主 IDE 自己提供执行隔离。纯 CLI 模式下 sandbox 弱。
3. **memory drift 靠 prompt 兜底**：相比 OpenClaw 的 temporal decay 加 retrieval，少了一层机制兜底。
4. **trace 较弱**：没有 Codex 风格的 rollout-trace.jsonl 全 session 回放。

## §5 · 5 件最值得借鉴的工程动作

1. **skillify 4 轮 AskUserQuestion 流程**（`src/skills/bundled/skillify.ts`）：R1 收 name 和 description，R2 收 step 和 arg，R3 拆解步骤，R4 抽 trigger。把「agent 自产 skill」拆成 4 个有边界的问题。
2. **`/security-review` 命令的 prompt 设计**（`src/commands/security-review.ts`）：senior security engineer 角色加 5 类漏洞加排除清单。
3. **MemoryType 加 2 种 prompt mode 矩阵**（`src/memdir/`）：4 types 乘 `{COMBINED, INDIVIDUAL}` 共 8 种组合。
4. **autoMode classifier 让 LLM 审用户规则**（`src/cli/handlers/autoMode.ts`）：规则审计也变成 LLM 任务。
5. **17 个 bundled skill 的边界感**：`loop`、`stuck`、`remember`、`verify` 都有明确退出条件，可以直接借鉴设计模式。
